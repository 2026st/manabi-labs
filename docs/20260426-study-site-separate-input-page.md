# 勉強サイト入力画面の分離

## 変更内容

勉強サイトページの入力UIを別ページへ分離した。

- 一覧ページ `app/routes/study.tsx`
  - 入力欄を削除
  - `サイトを追加` ボタンを追加し、`/study/new` へ遷移
  - 登録済みサイト一覧を表示
- 入力ページ `app/routes/study.new.tsx` を新規追加
  - サイト名・URL入力欄
  - `追加する` / `キャンセル` ボタン
- 共有ロジック `app/lib/study-sites.ts` を新規追加
  - 既定サイト（過去問道場）を保持
  - 追加サイトは `localStorage` 保存
  - URLの正規化と簡易バリデーションを実施
- ルーティング `app/routes.ts`
  - `/study/new` を追加

## 意図（なぜ必要か）

一覧閲覧と入力操作を分離し、画面の責務を明確化するため。
ユーザーは「見るページ」と「追加するページ」を行き来しやすくなる。

## 選択理由

- KISS: ルート分離 + 既存スタイル流用の最小変更
- YAGNI: 未要求の編集・削除・サーバ永続化は未実装
- DRY: サイト管理を `study-sites.ts` に集約

## 具体例

1. `勉強サイト` で `サイトを追加` を押す  
2. `/study/new` で `IPA` と `www.ipa.go.jp` を入力して `追加する`  
3. `勉強サイト` に戻ると `https://www.ipa.go.jp` が一覧に表示される
