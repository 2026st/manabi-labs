## 変更内容

- `worker.mjs` の SPA フォールバック判定を修正しました。
- 変更前は `url.pathname.includes(".")` でファイル判定していたため、`/article/<id>` の `<id>` にドットが含まれると HTML フォールバックされず、意図しない遷移の原因になっていました。
- 変更後は `lastSegment` のみを見て `\.[a-zA-Z0-9]+$` に一致する場合だけファイル扱いにしています。

## 実装意図（なぜ必要か）

- `article.$id` ルートをリロードしたときに、動的 ID にドットが含まれても SPA ルーティングを維持するためです。
- ルート URL を壊さずに `index.html` を返すことで、React Router が正しい画面 (`/article/:id`) を再描画できます。

## 具体例

- 変更前: `/article/2026.04.26` をリロード -> ファイル扱いされてフォールバック失敗
- 変更後: `/article/2026.04.26` をリロード -> `index.html` フォールバック -> `article.$id` がそのまま表示
